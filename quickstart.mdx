Get your SameVibe instance up and running quickly with this streamlined guide.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js 18 or higher** - [Download here](https://nodejs.org/)
- **PostgreSQL** - [Download here](https://www.postgresql.org/download/)
- **Git** - [Download here](https://git-scm.com/)

<Note>
  You can use cloud PostgreSQL services like [Supabase](https://supabase.com/), [Neon](https://neon.tech/), or [Railway](https://railway.app/) for easier setup.
</Note>

## Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the SameVibe repository from GitHub:
    
    ```bash
    git clone https://github.com/jupaorba/same-vibe.git
    cd same-vibe
    ```
  </Step>

  <Step title="Install Dependencies">
    Install the required npm packages:
    
    <CodeGroup>
    ```bash npm
    npm install
    ```
    
    ```bash yarn
    yarn install
    ```
    
    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>
  </Step>

  <Step title="Configure Environment Variables">
    Create a `.env` file in the root directory:
    
    ```bash
    DATABASE_URL="postgresql://username:password@localhost:5432/samevibe?schema=public"
    ```
    
    <Info>
      Replace `username`, `password`, and `localhost:5432` with your PostgreSQL credentials and host.
    </Info>
  </Step>

  <Step title="Set Up the Database">
    Generate the Prisma client and push the schema to your database:
    
    ```bash
    npx prisma generate
    npx prisma db push
    ```
    
    <Tip>
      Run `npx prisma studio` to open a visual database editor in your browser.
    </Tip>
  </Step>

  <Step title="Start the Development Server">
    Launch the Next.js development server:
    
    ```bash
    npm run dev
    ```
    
    The application will be available at [http://localhost:3000](http://localhost:3000)
  </Step>
</Steps>

## First Steps

Once your server is running, here's what to do next:

### 1. Create an Account

Navigate to [http://localhost:3000/signup](http://localhost:3000/signup) and create your first user account:

```typescript
// User model from prisma/schema.prisma
model User {
  id           Int      @id @default(autoincrement())
  email        String   @unique
  password     String   // bcrypt hashed
  name         String?
  sessionToken String?  @unique
  posts        Post[]
}
```

### 2. Log In

After creating an account, log in at [http://localhost:3000/login](http://localhost:3000/login). The authentication system will:
- Verify your password using bcrypt
- Create a session token
- Set secure HTTP-only cookies

### 3. Create Your First Post

Navigate to `/posts/new` and create your first post. Posts include:
- **Title** - A short, catchy headline
- **Content** - Your story, anecdote, or experience

```typescript
// Post model from prisma/schema.prisma
model Post {
  id        Int      @id @default(autoincrement())
  title     String
  content   String?
  published Boolean  @default(false)
  authorId  Int
  author    User     @relation(fields: [authorId], references: [id])
}
```

### 4. Explore the Feed

Visit `/posts` to see all posts from the community. The feed displays:
- Post titles and content
- Author information
- Interactive UI with glassmorphism effects

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database Connection Error">
    If you see `Error: Can't reach database server`, check that:
    - PostgreSQL is running on your system
    - Your `DATABASE_URL` is correct in the `.env` file
    - The database exists and is accessible
    
    Test your connection:
    ```bash
    npx prisma db pull
    ```
  </Accordion>

  <Accordion title="Prisma Client Not Generated">
    If you see `Cannot find module '@prisma/client'`, run:
    ```bash
    npx prisma generate
    ```
    
    This generates the Prisma client based on your schema.
  </Accordion>

  <Accordion title="Port Already in Use">
    If port 3000 is already in use, you can specify a different port:
    ```bash
    PORT=3001 npm run dev
    ```
  </Accordion>

  <Accordion title="Session/Cookie Issues">
    If you're having trouble staying logged in:
    - Clear your browser cookies for `localhost:3000`
    - Check that cookies are enabled in your browser
    - Ensure you're not in incognito/private mode (cookies persist differently)
  </Accordion>
</AccordionGroup>
